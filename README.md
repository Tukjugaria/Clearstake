# VCEquityNote

> 한국 벤처투자 관행·세법에 기반한 **SAFE·캡테이블·우선주 Exit Waterfall·스톡옵션/엔젤/RSU 세제·런웨이·투자수익** 계산기 모음.
> 창업자와 투자자 모두의 관점으로 시뮬레이션하는 **무료 공개 웹 도구**.

**모든 계산은 브라우저에서만 수행되며, 입력값은 서버로 전송되지 않습니다.** (서버·DB·외부 API·추적 스크립트 없음)

---

## 핵심 가치

- **양면 관점** — 동일 데이터를 창업자(희석·세금)와 투자자(전환·지분) 관점으로 전환해 보여줍니다.
- **한국 특화** — 벤처투자법상 SAFE 요건과 조세특례제한법 스톡옵션 세제 등 한국 제도를 반영합니다.
- **프라이버시** — 100% 클라이언트 사이드. 데이터는 브라우저를 떠나지 않습니다.

## 도구

도구는 `src/tools.ts` 레지스트리로 관리되며, 항목을 추가하면 네비게이션·홈에 자동 반영됩니다.

**지분·딜**
| 도구 | 설명 |
|---|---|
| SAFE 전환 계산기 | cap·discount 기반 전환 주식수·지분율, 적용 기준 표시 |
| 캡테이블 & 희석 시뮬레이터 | 라운드별 신규 발행·옵션풀(pre/post)·SAFE 전환 반영, before/after·희석 추이 |
| 통합 시나리오 | "이 라운드 후 옵션을 행사하면 세금은?" — 라운드 주당가치를 행사 시 시가로 연결 |
| 베스팅 스케줄 | 클리프 + 주기 베스팅 기준 현재 베스팅 수량·비율·향후 일정 |
| 목표 지분 역산 | 목표 지분을 지키기 위한 라운드 최소 pre-money 역산 |
| 전환사채(CB) vs SAFE | 이자가 붙는 CB와 무이자 SAFE의 전환 주식수·지분율 비교 |
| 우선주(RCPS) Exit Waterfall | 청산우선권·참가·전환을 반영한 엑싯 매각대금 분배 + 엑싯가치별 추이 |
| 텀시트 비교 | 두 텀시트(밸류·옵션풀·청산우선권)의 지분 희석·엑싯 회수금을 나란히 비교 |

**투자·수익**
| 도구 | 설명 |
|---|---|
| 투자 수익률 (MOIC·IRR) | 진입가·지분·엑싯 가치로 회수금·배수·IRR·pro-rata 투자금 |
| 밸류에이션 추정 | 매출·순이익 멀티플로 기업가치를 개략 추정하고 범위 제시 |
| 펀드 성과지표 (TVPI·DPI·XIRR) | 납입·분배·NAV로 TVPI/DPI/RVPI + 불규칙 현금흐름 XIRR |
| 목표 수익 역산 (Fund-returner) | 펀드를 N배 돌려주기 위한 필요 엑싯 기업가치 역산 |

**운영**
| 도구 | 설명 |
|---|---|
| 런웨이 · 번레이트 | 현금·월손익으로 소진 시점(런웨이)·손익분기·펀딩 데드라인 |

| 번 멀티플 (자본효율) | 순현금소모 ÷ 순신규 ARR, 효율 등급 |
| SaaS 지표 (LTV·CAC·NRR) | LTV·CAC·회수기간·NRR/GRR 등 단위경제·유지율 |
| 손익분기점(BEP) | 고정비 회수 판매 수량·매출, 안전마진 |

| RSU 세제 계산기 | RSU 베스팅 시 근로소득 + 매각 시 양도소득 과세 개략 추정 |

## 기술 스택

- **Vite + React + TypeScript** (정적 SPA, 무료 호스팅 가능)
- **Tailwind CSS v4** (스타일)
- **Recharts** (희석·런웨이·베스팅·Waterfall 시각화 — 차트가 있는 페이지에서만 지연 로드)
- **React Router** (HashRouter — 별도 서버 설정 없이 어디서나 동작)
- **Vitest** (계산 로직 단위테스트)

## 개발

```bash
npm install      # 의존성 설치
npm run dev      # 개발 서버 (http://localhost:5173)
npm test         # 단위테스트
npm run build    # 프로덕션 빌드 (dist/)
npm run preview  # 빌드 결과 미리보기
```

## 배포

순수 정적 사이트이므로 빌드 산출물(`dist/`)을 무료 정적 호스팅에 올리면 됩니다.

- **Vercel / Netlify** — 프레임워크 자동 감지, `npm run build` / 출력 `dist`
- **GitHub Pages** — `dist/`를 배포. HashRouter를 사용하므로 SPA 리라이트 설정이 필요 없습니다.

## 법령 의존 값 관리 (`taxConfig`)

세율·한도·일몰연도 등 **법령에 의존하는 값은 코드에 하드코딩하지 않고** 단일 모듈로 분리합니다.
개정 시 이 파일들만 수정하면 됩니다.

- `src/config/taxConfig.ts` — 스톡옵션 비과세 한도(연/누적)·일몰·분할납부, 종합소득세율, 양도세율(소액주주/대주주), 엔젤투자 소득공제 구간, 법인세율, 창업 세액감면 프리셋
- `src/config/safeConfig.ts` — 벤처투자법 SAFE 요건 체크리스트

각 수치에는 **출처·기준일** 주석을 달며, 불확실한 값은 `TODO(검증)`으로 표시합니다.
화면 하단의 `lastUpdated` 기준일과 출처 목록은 config에서 자동으로 노출됩니다.

### 제도적 근거

- **SAFE(조건부지분인수계약)** — 벤처투자 촉진에 관한 법률 시행규칙 제3조 (2020.8 시행), 중소벤처기업부 표준계약서
- **스톡옵션 비과세·납부특례·양도세 과세선택** — 조세특례제한법 제16조의2~4
- **엔젤투자 소득공제** — 조세특례제한법 제16조
- **창업·벤처기업 세액감면** — 조세특례제한법 제6조
- **종합소득세율** — 소득세법 제55조 / **주식 양도소득세** — 소득세법 제103·104조 / **법인세율** — 법인세법 제55조

> ⚠️ 일몰연도·연간 한도 등은 개정될 수 있으므로 배포 전 최신 법령을 재확인하고 `taxConfig`를 갱신하세요.
> 보유기간·대주주 요건 등 일부 변수는 미반영한 **개략 추정**입니다.

## 면책

본 도구는 **일반 정보 제공용**이며 법률·세무·투자 자문이 아닙니다. 계산은 단순화된 모델에 기반한
**개략 추정**이며, 실제 의사결정은 반드시 전문가(변호사·세무사) 확인이 필요합니다.

## 프로젝트 구조

```
src/
├── lib/             # 순수 계산 엔진 (각 폴더에 *.ts + *.test.ts)
│   ├── safe/        # SAFE 전환
│   ├── captable/    # 캡테이블 & 희석 시뮬레이션
│   ├── waterfall/   # 우선주(RCPS) Exit Waterfall
│   ├── termsheet/   # 텀시트 비교 (captable·waterfall 조합)
│   ├── convertible/ # 전환사채(CB) vs SAFE
│   ├── target/      # 목표 지분 역산
│   ├── vesting/     # 베스팅 스케줄
│   ├── returns/     # 투자 수익률(MOIC·IRR)
│   ├── valuation/   # 밸류에이션 추정
│   ├── runway/      # 런웨이·번레이트
│   ├── tax/         # 스톡옵션·엔젤·창업감면·RSU 세제 + 소득세/양도세 공용 유틸
│   ├── format.ts    # ₩/% 포맷팅·파싱
│   ├── groups.ts    # 주주 그룹 색상·라벨
│   └── scenarioUrl.ts # 시나리오 URL 인코딩(쿼리스트링)
├── config/          # 법령 의존 값 (taxConfig, safeConfig)
├── hooks/           # 헤드리스 계산 훅 (상태 ↔ 엔진)
├── context/         # 관점(창업자/투자자) 전역 상태
├── components/      # UI (layout, ui, charts)
├── tools.ts         # 도구 레지스트리 (네비·홈 자동 생성)
└── pages/           # 홈 + 각 계산기 페이지 + FAQ
```
